# CARCH

CARCH는 사용자가 이미 보유한 신용카드 중에서 결제 예정 정보와 카드 혜택 조건을 비교해 최적의 카드를 추천하는 보유 카드 운영 최적화 서비스입니다.

## 핵심 목표

- 신규 카드 발급 추천보다 보유 카드 사용 최적화에 집중합니다.
- 사용자가 결제 예정 가맹점, 카테고리, 금액을 입력하면 보유 카드 중 가장 유리한 카드를 추천합니다.
- 추천 결과에는 예상 혜택 금액, 전월 실적 충족 여부, 남은 혜택 한도, 예산 초과 여부를 함께 제공합니다.

## MVP 범위

- 회원가입 및 로그인
- 카드명 유사 검색
- 보유 카드 등록
- 카드별 실적 및 혜택 한도 조회
- 소비 내역 직접 입력
- 결제 예정 정보 기반 카드 추천
- 추천 사유 표시
- 월 예산 및 카테고리별 예산 관리
- 간단한 월간 리포트

## 백엔드 연동 — 브랜치 `feat/backend-integration` (2026-06-25~)

소비계획 페이지를 실제 백엔드와 단계적으로 연동한다. **원칙: `main`(현재 프론트)은 절대 안 건드리고
별도 브랜치에서 작업, 모든 연동은 "백엔드 우선 + 실패 시 기존 목 데이터로 폴백" → 화면은 기존과 100% 동일하게 유지.**
각 단계 완료마다 이 로그에 적고 브랜치에 푸시한다. 검증 후 `main`에 머지 예정.

전체 계획(우선순위): ①카드 현황 ②AI 카드 추천 ③목표·예산 DB 저장 ④할인 한도 정확 계산 ⑤5개월 히스토리 백엔드화.

### ① 카드 현황 실연동 — 완료
- `BudgetListView`의 카드 현황을 `mockData` 직접 import → **`fetchOwnedCards()` + `fetchTransactions()`** 기반으로 전환.

다만 **카드가 없는 사용자에게 데모 카드를 자동 시드**해서 신규 가입자도 데모 카드로 시작하던 문제를 바로잡았다.
- `is_demo_seed_user(user)` 추가: 이메일이 데모(`demo@carch.local`)·dev-admin일 때만 True.
- `ensure_owned_cards_seeded` / `ensure_transactions_seeded` / `ensure_purchase_plans_seeded` 에 게이트 추가 → **데모/시연 계정만 자동 시드, 실제 신규 가입자는 빈 지갑/빈 거래로 시작**(본인이 셀프 등록).
- 검증: 새 이메일 회원가입 → `GET /api/owned-cards/` = `{count:0, results:[]}`(빈 지갑). 시연(dev-admin)은 데모 카드 3장 그대로.

### ★ 검토·보수 — 신규 유저 화면 정확성
- **버그 수정**: AI 추천이 "백엔드 빈 결과"를 "호출 실패"와 혼동해 신규 유저에게 **데모 추천을 잘못 노출**하던 문제. → `aiBundleLoaded` 플래그로 분리: 응답 성공(빈 결과 포함)이면 백엔드 결과만 사용(보유 카드 없으면 빈 추천 → 섹션 자동 숨김), 호출 실패 시에만 로컬 폴백.
- 토큰 교체로 신규 유저 화면 직접 검증: 카드 현황·AI 추천 섹션 자동 숨김, 이번 달 사용 0원, 5개월 차트 본인 데이터(전부 0) → **남주현 데이터 일절 노출 안 됨**. 데모(dev-admin)는 카드 3·AI 3·1,051,950원 그대로.

### ④ 할인 한도 정확 계산 — 완료
- 카드 현황의 "받은 할인"을 하드코딩 상수 → **이번 달 사용액 × 카드 혜택률**(`benefitItems[0].ratePercent`)을 월 한도 내에서 계산(`min(spent×rate, discountLimit)`).
- 혜택률/한도 정보가 없으면(예: 신규 사용자가 등록한 카드) 0으로 → 빈 지갑/신규 유저는 `0원 / 0원`.
- 이번달 할인 미적용(전월 실적 미충족) 카드는 그대로 `0원 / 0원`.
- 검증: 데모 LIKIT = `6,055원 / 30,000원`(=121,100×5%, 바 20% 시각 동일), 미적용 카드 0/0.

이날은 소비계획(`frontend/src/views/BudgetListView.vue`) 한 화면을 집중적으로 다듬었다.
모든 변경은 프론트엔드 한정이며 백엔드는 건드리지 않았다 — 이 페이지는 아직 `mockData`/`localStorage` 기반이고,

### 카드 현황 (구 "카드 사용 가이드" 재설계)
- 섹션 제목을 **"카드 현황"** 으로 변경.
- 보유 카드별로 **사용 한도 진행바 + 할인 한도 진행바** 표시.
- **전월 실적 기반 할인 가능 여부**를 플래그로 분리: `이번달 할인 ✓/✗`(전월 실적 충족 여부) + `실적 충족 ✓ / 실적 NN%`(이번달 실적 진행). 이번달·다음달 상태를 동시에 보이게 함.
- 실적 미충족 카드는 배경을 연한 빨강, 가능 카드는 연한 파랑으로 구분(테두리 없음).
- 카드명 아래 브랜드(VISA 등) 대신 **짧은 혜택**(예: `쇼핑 10%`) 표기.
- 이번달 할인이 안 되는 카드는 **할인 한도 `0원 / 0원`** 으로 표기.
- 데이터: `mockData.js` 카드에 `discountLimit`/`discountUsed` 추가.

### 이번 달 사용 헤더
- 하단에 **목표 금액** 행 + 연필 버튼으로 인라인 편집(저장은 `budgetStorage` → localStorage, 분석/예산 화면과 동일 저장소 연동).
- 스크롤 내리면 **목표 금액 행만 접히고** 이번 달 사용 요약은 상단 고정(반응형).
- 헤더 박스 테두리 제거: 원인은 `app.css`의 전역 `!important` 규칙이 헤더 내부 `a`(=`.cs-main`)에 1px 테두리를 강제하던 것 → `.cs-main` 전용 오버라이드로 해제.

### AI 카드 추천 섹션 (신설)
- 까치 마스코트(`/card-images/magpie-face2.png`) + **말풍선**으로 추천 한마디(예: "식비엔 LOCA LIKIT Eat가 딱이에요!").
- 카테고리별 추천 카드 + **정확한 혜택 문구**(`식비 6% 할인` 등) + 예상 절약액/최대 절약액.
- 배경에 은은한 하늘색→연보라 그라데이션(AI 느낌).
- 헤더 옆 **"자세히 보기"(돋보기)** → 기존 AI 사용 추천 페이지 `/recommendations/usage`.
- 추천 로직은 기존 로컬 `cardGuideItems`(남은 예산 + 카드 실적) 재사용 — 아직 백엔드 AI 아님.

### 나의 예산 (월별 시각화)
- 월별 카드 목록 → **최근 5개월 막대그래프**로 교체: 예산(연회색 트랙) + 사용(색상) 겹침, 초과월 빨강, 현재월 강조.
- 아래 **5개월 표**(월 / 사용 / 예산 / 잔액), 잔액 `+` 초록 / `-` 빨강.
- **각 달(막대·표 행) 클릭 → 그 달 소비 분석**(`/analytics?month=YYYY-MM`)으로 이동.
- 막대그래프 과거 달 사용액은 분석 화면과 동일한 데모(`monthlyAnalytics.js`)에서 가져와 **두 화면 숫자를 일치**시킴(6월은 현재 예산 데이터).
- `AnalyticsView`가 `?month=` 쿼리를 읽도록 추가(`useRoute` + 진입 시/변경 시 적용) → 클릭한 달의 분석이 실제로 표시됨(기존엔 항상 6월 기본).

### 백엔드 연동 현황 (상의 메모)
- 소비계획 페이지만 `mockData`를 직접 import해 백엔드와 단절. 나머지 탭은 `api.js`(API 호출 + 목 폴백) 사용.
- 이미 있는 것(재사용 가능): `/api/owned-cards/`, `/api/transactions/`, `/api/recommendations/cards/`(AI).
- 신규 필요: 목표 금액·카테고리 예산 저장용 Budget 모델/API(현재 localStorage), 할인 한도 정확 계산, 과거 5개월 거래 시드.
